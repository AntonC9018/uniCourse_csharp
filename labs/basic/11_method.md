# Лабораторная работа 11

- Тема: **instance методы**.
- [Видео](https://www.youtube.com/watch?v=UVLUbfcCcG0&list=PL4sUOB8DjVlVVw9Yx_tUO7fRPDYeaACXD&index=8&t=822s), 

## Идеи

- Член класса
- Instance поля
- Instance класса (экземпляр)
- Статический метод
- Instance метод

## Задание

1. Создайте *абстракцию* (класс) с публичными полями для библиотеки, 
   если таковой еще нет.
   Она нужна для того, чтобы было к чему привязывать методы.

   <details>
   <summary>Подсказка</summary>

   Имеется ввиду класс с полем массива книг.
   </details>

2. Сделайте поля `required`, чтобы удовлетворить гарантии nullability.

3. Переделайте функции операций с книгами на функции,
   принимающие параметром абстракцию.

4. Сделайте из функий методы.

   <details>
   <summary>Как?</summary>

   Цель — вызов функции как `Functions.BorrowBook(library, "123")`
   сменить на `library.BorrowBook("123")`.

   1. Меняем синтаксис *вызова* этой функции, везде в коде, как указано выше.
   2. Переносим *определение* функции из `Functions` в `Library` (класс абстракции).
   3. Стираем `static`.
   4. Стираем первый параметр.
   5. Меняем все упоминания первого параметра на `this`.
   </details>

   <details>
   <summary>Зачем?</summary>

   Пока преимуществ нет — код будет эквивалентен процедуральному (как было).
   В следующих лабах будет про инкапсуляцию, где это надо.
   </details>
